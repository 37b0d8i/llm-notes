Simple Build a simple text-in → text-out Typescript codes

LLM API + CLI Typescript codes

Prerequisites

  • repo exists locally and on GitHub
  • Bun runtime installed on your machine
  • You have an API key from your chosen LLM provider

Part A — Prepare the project

Step 1: Navigate to project root

cd ts-ai-systems

Since Bun natively supports TypeScript and uses isolated global runtimes, you do not need to create or activate a traditional Python virtual environment.

Step 2: Initialize project and install dependencies

Initialize a fresh Bun project structure if you haven't already:

bun init -y

You will need the official OpenAI community or vendor SDK. Bun reads environment variables natively from .env files out of the box:

bun add openai

Step 3: Create an environment file for secrets

In the project root, create a file named .env and add:

OPENAI_API_KEY=your_api_key_here

Confirm .env is in your .gitignore file. If not, add it:

.env
node_modules/
Never commit your .env file.

Part B — Call an LLM API from TypeScript

Step 4: Create a new script file

Create the scripts target directory and file:

mkdir -p scripts
touch scripts/cli_assistant.ts

Step 5: Paste this code (LLM API call + CLI loop)

Open scripts/cli_assistant.ts and add the following codebase:

import OpenAI from "openai";
import readline from "readline";

// Bun natively injects process.env from your .env file
const apiKey = Bun.env.OPENAI_API_KEY;

if (!apiKey) {
  console.error("Error: Missing OPENAI_API_KEY. Put it in a .env file.");
  process.exit(1);
}

const client = new OpenAI({ apiKey });

function buildMessages(userText: string) {
  return [
    {
      role: "system" as const,
      content: "You are a helpful assistant. Be concise, correct, and do not invent facts. If unsure, say you are unsure."
    },
    { 
      role: "user" as const, 
      content: userText 
    }
  ];
}

async function callLlm(userText: string): Promise<string> {
  const messages = buildMessages(userText);

  const resp = await client.chat.completions.create({
    model: "gpt-4o-mini",
    messages: messages,
    temperature: 0.2,
    max_tokens: 300,
  });

  return resp.choices[0].message.content?.trim() || "";
}

async function main() {
  console.log("CLI AI Assistant (type 'exit' or 'quit' to quit)");

  // Setup readable stream reader interface for terminal IO
  const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout,
  });

  const askQuestion = () => {
    rl.question("\nYou: ", async (input) => {
      const userText = input.trim();

      if (userText.toLowerCase() === "exit" || userText.toLowerCase() === "quit") {
        console.log("Goodbye.");
        rl.close();
        process.exit(0);
      }

      if (!userText) {
        console.log("Please enter a question.");
        askQuestion();
        return;
      }

      try {
        const answer = await callLlm(userText);
        console.log("\nAssistant:", answer);
      } catch (e: any) {
        console.error("\nError calling the LLM API:", e.message || e);
      }

      askQuestion(); // Core interface loop recursion
    });
  };

  askQuestion();
}

main();

Notes:

  • Uses TypeScript structural types and native asynchronous promises.
  • Uses Bun's built-in Bun.env utility directly.
  • Low temperature setting for consistent deterministic response validation.

Step 6: Run the script

Execute the script immediately via Bun runtime without manual compilation phases:

bun run scripts/cli_assistant.ts

Try prompts like:

  • “Explain tokens in one paragraph.”
  • “Rewrite this sentence more formally: …”
  • “Give me a 3-bullet summary of: …”

Troubleshooting

  • "Missing OPENAI_API_KEY"
    Ensure your local .env file rests in the absolute execution root of the project directory.
  • "Command not found: bun"
    Ensure your global shell environments possess local environment binary context mappings for Bun paths.
