Git best practices

Stop wandering in the jungle. Use a simple, predictable workflow and master just 3 commands.

Core principle

Simple branch strategy (main + feature branches) → Small, frequent commits → Only 3 undo commands → Always visualize before acting.

The 3 commands you need

checkout

git checkout file.js

Discard local changes to one file.

Safe

reset

git reset HEAD~1

Undo last commit, keep changes locally.

Medium risk (before push)

revert

git revert HEAD

Undo a commit (after push). Creates a new commit.

Safe

Never use `git reset --hard` unless you fully understand it. Use `revert` instead (safer for shared history).

Your daily workflow

$ git status — Always check first
$ git add file.js — Stage your work
$ git commit -m "fix: …" — Commit with clarity
$ git push — Push to remote
$ git log --oneline -5 — See what happened

This covers 95% of real development work.

Decision tree: What to do when things go wrong

I changed a file and want to discard changes

→ Use `git checkout file.js`

I committed but haven't pushed yet

→ Use `git reset HEAD~1` (edit files, recommit)

I pushed a bad commit to shared branch

→ Use `git revert HEAD` (safest for shared history)

Why this works

  • Predictability: Use 5–6 commands every day. No surprises.
  • Visibility: Always check status and history before acting.
  • Safety: Use `revert`, not `reset --hard`, for shared branches.
  • Simplicity: Feature branches + commits + pull requests. A clear path.
